# Simple Web Development Portfolio

I built a lightweight, responsive **three-page mini website** for CDV05 to showcase my frontend development skills, biography, and academic coding projects to my prospective employers, as was required of us. 

## 📑 Project Structure

The website consists of three interconnected static pages:
* **Page 1: About Me (`index.html`)** - The landing page introducing myself, my career goals, and my current skill set.
* **Page 2: Projects (`projects.html`)** - A showcase of my work featuring clickable, direct links to my GitHub repositories.
* **Page 3: Contact (`contacts.html`)** - A clean page providing my professional contact information and social profiles.

## 🛠️ Built With

* **HTML5** - Used to structure the semantic layout and navigation of the application.
* **CSS3** - Used to style custom layouts, typography, color palettes, and responsive elements.

## 🚀 Launching the Project Locally

Because this project is currently self-contained and **not yet deployed live**, you can launch and preview the application directly from the project files on your computer:

1. **Download or Clone** the project folder to your local machine.
2. Locate the root directory of the folder.
3. Find the main file named **`index.html`** (this is the designated homepage).
4. **Double-click** `index.html` to launch it immediately in your default web browser (such as Chrome, Edge, or Safari).
5. Use the navigation menu bars within the browser window to seamlessly click through to the *Projects* and *Contact* pages.

## 🌐 Future Deployment Plans

When this mini site is ready to go live on the public internet, it will be hosted using **GitHub Pages** due to its seamless integration with static HTML/CSS files. 

### How to Deploy via GitHub Pages:
1. Push this complete project folder into a public repository on your personal [GitHub Account](https://github.com).
2. Ensure your homepage file remains exactly named **`index.html`** in the root directory.
3. On GitHub, navigate to the **Settings** tab of your online repository.
4. Scroll down to the **Pages** menu item on the left-hand sidebar.
5. Under the *Build and deployment* source section, switch the branch configuration from *None* to **`main`** (or `master`).
6. Click **Save**. GitHub will automatically process your static files and generate a live, public URL link within a few minutes.
